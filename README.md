# Система учёта инцидентов техподдержки

Десктопное приложение на **C# / WinForms / .NET 6 / SQLite**.

## Требования

- .NET 6 SDK (Windows): https://dotnet.microsoft.com/download/dotnet/6.0
- Windows 10/11

## Сборка и запуск

```bash
cd IncidentTracker
dotnet restore
dotnet build
dotnet run
```

Или публикация в один исполняемый файл:
```bash
dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true
```

Файл появится в `bin\Release\net6.0-windows\win-x64\publish\IncidentTracker.exe`.

## Функциональность

| Модуль            | Что делает |
|-------------------|-----------|
| **Регистрация**   | Создание инцидента с авто-номером `INC-YYYYMM-NNNN`, автором, подразделением, категорией, приоритетом, описанием |
| **Обработка**     | Назначение исполнителя, смена статуса, добавление комментариев |
| **SLA**           | Автоматический расчёт срока реакции и решения по приоритету; цветовая индикация (красный/жёлтый/зелёный) |
| **Файлы**         | Прикрепление, просмотр и удаление файлов (копируются в папку `Attachments/<id>/`) |
| **История**       | Каждое изменение статуса и действие пишется в таблицу History |
| **Отчёты**        | По статусам, по категориям, список просроченных; экспорт в CSV |
| **Журнал**        | Все действия пользователя записываются в `app.log` |

## Статусы инцидентов

```
Новый → В работе → Ожидает информации → Решён → Закрыт
                                      ↘ Отклонён
```

## SLA по приоритетам

| Приоритет   | Реакция | Решение |
|-------------|---------|---------|
| Критический | 1 ч.    | 4 ч.    |
| Высокий     | 4 ч.    | 24 ч.   |
| Средний     | 8 ч.    | 72 ч.   |
| Низкий      | 24 ч.   | 168 ч.  |

## База данных

SQLite-файл `incidents.db` создаётся автоматически в папке запуска.
Таблицы: `Incidents`, `History`, `Comments`, `Attachments`.
